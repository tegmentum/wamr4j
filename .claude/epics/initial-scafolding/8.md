# Task 002: Java Module Scaffolding

## Summary
Create comprehensive module structure for all 6 Maven modules with proper POM configurations, package hierarchies following ai.tegmentum.wamr4j pattern, and foundational interfaces and classes that define the unified API architecture.

## Context
With the root project structure established, this task creates the detailed module scaffolding that will serve as the foundation for all subsequent development. This includes setting up proper module dependencies, package structures, and placeholder implementations that establish the architectural boundaries between public APIs and private implementations.

## Acceptance Criteria

### Module POM Files
- [ ] Create `pom.xml` for `wamr4j` module (public API) with no implementation dependencies
- [ ] Create `pom.xml` for `wamr4j-jni` module with JNI-specific dependencies
- [ ] Create `pom.xml` for `wamr4j-panama` module with Panama FFI dependencies (Java 23+)
- [ ] Create `pom.xml` for `wamr4j-native` module with Rust build configuration
- [ ] Create `pom.xml` for `wamr4j-benchmarks` module with JMH dependencies
- [ ] Create `pom.xml` for `wamr4j-tests` module with comprehensive test dependencies
- [ ] Configure proper module interdependencies (tests depend on implementations, implementations depend on native, etc.)
- [ ] Set appropriate Java version targets per module (17 for most, 23+ for Panama)

### Package Structure Creation
- [ ] Create `ai.tegmentum.wamr4j` package structure in wamr4j module
- [ ] Create `ai.tegmentum.wamr4j.jni` package structure in wamr4j-jni module  
- [ ] Create `ai.tegmentum.wamr4j.panama` package structure in wamr4j-panama module
- [ ] Create standard Maven directory layout: `src/main/java`, `src/test/java`, `src/main/resources`
- [ ] Create `src/main/rust` directory in wamr4j-native module
- [ ] Establish proper package hierarchy with internal/impl subpackages for private APIs

### Core Interface Definitions (wamr4j module)
- [ ] Create `WebAssemblyRuntime` interface - main entry point for WebAssembly operations
- [ ] Create `WebAssemblyModule` interface - represents compiled WebAssembly modules
- [ ] Create `WebAssemblyInstance` interface - represents instantiated WebAssembly modules
- [ ] Create `WebAssemblyFunction` interface - represents callable WebAssembly functions
- [ ] Create `WebAssemblyMemory` interface - represents WebAssembly linear memory
- [ ] Create `RuntimeFactory` class - factory for creating runtime instances with auto-detection
- [ ] Create exception hierarchy: `WebAssemblyException`, `CompilationException`, `RuntimeException`, `ValidationException`

### Implementation Module Scaffolding
- [ ] Create placeholder implementation classes in wamr4j-jni implementing core interfaces
- [ ] Create placeholder implementation classes in wamr4j-panama implementing core interfaces
- [ ] Set up proper module isolation - implementations not visible to public API consumers
- [ ] Create service provider configuration files for runtime discovery
- [ ] Establish internal utility classes for common functionality

### Native Module Structure
- [ ] Create Cargo.toml for Rust project with proper WAMR dependencies
- [ ] Set up cross-compilation targets for all supported architectures
- [ ] Create placeholder Rust files with JNI and Panama FFI export functions
- [ ] Configure build integration with Maven using cargo-maven-plugin

### Test Module Foundation
- [ ] Create abstract test base classes for runtime-agnostic testing
- [ ] Set up test resource directories for WebAssembly test files

## Technical Implementation Details

### Module Dependency Graph
```
wamr4j (public API) 
├── No dependencies on implementation modules
└── Only standard Java libraries

wamr4j-jni (private implementation)
├── Depends on: wamr4j-native
└── Implements: wamr4j interfaces

wamr4j-panama (private implementation)  
├── Depends on: wamr4j-native
└── Implements: wamr4j interfaces

wamr4j-native (shared native library)
├── Rust project with WAMR bindings
└── Exports both JNI and Panama FFI functions

wamr4j-tests (integration tests)
├── Depends on: wamr4j, wamr4j-jni, wamr4j-panama
└── Test dependencies: JUnit 5, AssertJ, Testcontainers

wamr4j-benchmarks (performance tests)
├── Depends on: wamr4j, wamr4j-jni, wamr4j-panama  
└── JMH dependencies for microbenchmarking
```

### Core Interface Signatures
```java
// Public API interfaces in ai.tegmentum.wamr4j package
public interface WebAssemblyRuntime extends AutoCloseable {
    WebAssemblyModule compile(byte[] wasmBytes) throws CompilationException;
    WebAssemblyModule compile(Path wasmFile) throws CompilationException;
}

public interface WebAssemblyModule extends AutoCloseable {
    WebAssemblyInstance instantiate() throws RuntimeException;
    String[] getExportNames();
    String[] getImportNames();
}

public interface WebAssemblyInstance extends AutoCloseable {
    WebAssemblyFunction getFunction(String name) throws RuntimeException;
    WebAssemblyMemory getMemory() throws RuntimeException;
}
```

### Service Provider Configuration
- Create `META-INF/services/ai.tegmentum.wamr4j.spi.RuntimeProvider` files
- Configure automatic runtime detection based on Java version
- Implement fallback mechanism (Panama → JNI) for Java 23+

### Package Visibility Rules
- Public API: All interfaces and exceptions in `ai.tegmentum.wamr4j` are public
- Implementation: All classes in `ai.tegmentum.wamr4j.{jni,panama}.impl` are package-private
- Internal: All utility classes in `ai.tegmentum.wamr4j.{jni,panama}.internal` are package-private
